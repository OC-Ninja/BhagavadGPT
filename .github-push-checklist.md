# ✅ GitHub Push Checklist

Before pushing to GitHub, verify these items:

## Security Checks

- [ ] `.env` files are in `.gitignore`
- [ ] `.env.example` files are created (without real credentials)
- [ ] No API keys in code files
- [ ] No passwords or secrets in configuration files
- [ ] Google OAuth credentials are not committed

## Documentation

- [ ] README.md is complete and accurate
- [ ] SETUP.md has clear installation instructions
- [ ] LICENSE file is present
- [ ] All links in README work correctly

## Code Quality

- [ ] Backend runs without errors
- [ ] Frontend builds successfully
- [ ] Docker compose works
- [ ] No debug code or console.logs left in production code

## Files to Verify

Run these commands to check:

```bash
# Check what will be committed
git status

# Verify .env files are ignored
git check-ignore -v **/.env

# See what files will be added
git ls-files --others --exclude-standard
```

## Safe to Commit

These files SHOULD be committed:
- ✅ `.env.example` files
- ✅ `README.md`
- ✅ `SETUP.md`
- ✅ `LICENSE`
- ✅ `.gitignore`
- ✅ Source code files
- ✅ `requirements.txt`
- ✅ `package.json`
- ✅ `docker-compose.yml`
- ✅ `librechat.yaml`

## Should NOT be Committed

These files should be IGNORED:
- ❌ `.env` (actual environment files)
- ❌ `node_modules/`
- ❌ `venv/`
- ❌ `__pycache__/`
- ❌ `gita_knowledge_base/` (ChromaDB data)
- ❌ `meili_data*/`
- ❌ `logs/`
- ❌ API keys or secrets

## Final Check

Before pushing:

```bash
# Review all changes
git diff

# Check for sensitive data
git grep -i "api_key\|secret\|password" -- ':!*.example' ':!*.md'
```

If the grep command returns nothing, you're good to go!
